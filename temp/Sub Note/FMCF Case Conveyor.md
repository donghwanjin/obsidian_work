#### 3.2.1.4. DCI TU message flow at location types

By default, for each location where the PLC sends a TURP, TUDR, or TUEX message, WCS should send a TUMI in response to the same PLC. Therefore, it is simpler to specify exceptions to this.

Implementation in MH_CC_DCI_RoutingRequired

|Loc types where no TUMI should be sent|- MH_CC_LOC_TYPE_QA<br>- MH_CC_LOC_TYPE_TRAY_STACK_BUFFER<br>- MH_CC_LOC_TYPE_TRAY_STACKER<br>- MH_CC_LOC_TYPE_LINEAR_SORTER_SETD<br>- MH_CC_LOC_TYPE_LINEAR_SORTER_DIVERT<br>- MH_CC_LOC_TYPE_LINEAR_SORTER_RECIRC<br>- MH_CC_LOC_TYPE_DEPALL_CASE_TRAYED<br>- MH_CC_LOC_TYPE_DEPALL_PROFILE_CHECK<br>- MH_CC_LOC_TYPE_DEPALL_TRAY_REQUEST<br>- MH_CC_LOC_TYPE_MCAP_INFEED|
|Loc types where TUMI should be sent via scheduler / background process|- MH_CC_LOC_TYPE_DETRAYING_SPLITTER<br>- MH_CC_LOC_TYPE_MCAP_CALL_FORWARD|
|Loc types and locations where TUMI should be sent to a different PLC||
